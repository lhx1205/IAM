# 컴퓨터와 대화하는 두 가지 방법



#### 1\. GUI (Graphic user Interface)

* 사진이 있는 메뉴판을 보고 손가락으로 주문하는 방식
* 장점 : 쉽고 직관적 ex) 마우스로 아이콘을 클릭하는 윈도우, mac OS 화면



#### 2\. CLI (Command line Interface)

* 주방장에게 직접 말로 주문하는 방식
* 장점 : 원하는 대로 정확하고, 세밀함, 빠르게 요청 가능 (ex. 검은 화면에 명령어를 타이핑하여 소통하는 터미널, 명렁 프롬프트)



## 왜 CLI를 사용해야 하는가?

1. 효율성 : 키보드만으로 모든 작업을 빠르고 가볍게 수행
2. 정밀한 제어 : 그래픽 화면에 보이지 않는 숨겨진 설정까지 정밀하게 제어 가능
3. 강력한 자동화  : 단순하고 반복적인 작업을 명령어 한 줄로 자동화 가능
4. 표준 환경 서버, 클라우드 등 그래픽이 없는 개발 환경의 필수 도구



&#x20;CLI에서 마침표의 역할

1. 점 하나 - 현재 디렉토리
2. 점 두 개 - 부모 디렉토리



기초 문법

1. touch : 파일 생성
2. mkdir : 새 디렉토리 생성
3. ls : 현재 작업중인 디렉토리 내부의 폴더 / 파일 목록을 출력
4. cd : 현재 작업 중인 디렉토리를 변경(위치 이동)
5. start : 폴더/파일을 열기
6. rm : 파일 삭제 (디렉토리 삭제는 -r옵션을 추가 사용)
7. pwd : 현재 작업 중인 디렉토리의 절대 경로를 출력 

(처음\~ 파일명까지 오는 경로 - 절대 경로)

(윈도우\~현재까지 오는 경로 - 상대 경로)
